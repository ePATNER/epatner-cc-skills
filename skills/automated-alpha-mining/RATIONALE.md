# Rationale

**Status: draft, needs review before merging.**

## What it does

Guides running an LLM-driven or evolutionary alpha-mining process itself —
proposing candidate signal formulas at scale, mutating/recombining
promising ones, tracking search-process memory to avoid re-exploring dead
ends — with guardrails specifically aimed at the multiple-testing and
overfitting risk this kind of high-volume automated search creates, which
is larger than for manual, one-formula-at-a-time research.

## Why included

Found via fresh search after 23 skills were already shipped, and notably
different in kind from anything else in the repo: every other skill
addresses a single technique, model, or process; this one addresses the
meta-process of using LLM-driven/evolutionary search to generate many
candidate signals automatically. It's also unusually well-matched to this
operation's context specifically — the collaborator conducting this
research is itself an LLM, and 2026 research (evolutionary LLM alpha
mining frameworks, agentic factor-discovery systems) shows this is an
active, fast-moving area directly applicable here.

## Extent of expected help

The process can genuinely expand the volume of candidate signals explored
beyond what manual research allows, which is valuable for an operation
explicitly building many strategies/micro-strategies — but the skill is
deliberately weighted toward guardrails over capability description,
because the realistic risk is that this process multiplies the number of
spurious "discoveries" faster than it produces real ones, especially given
financial data's low signal-to-noise ratio. The explicit tie-ins to
`purged-cv-backtesting` and `quant-experiment-tracking` are the load-bearing
part of this skill — automated mining without those two disciplines
attached is a strictly worse version of a known failure mode (data
snooping), not a shortcut around it.

## Key risks / limitations

- The single biggest risk this skill addresses: treating a high internal
  score from the mining process's own search/scoring loop as validation.
  That score is generated by the same process being asked to overfit, and
  it is not independent evidence of a real effect.
- Automated mining can generate trial counts far too large to track
  manually; without disciplined logging (see `quant-experiment-tracking`)
  the multiple-testing correction needed to trust any result becomes
  impossible to apply correctly.
- Even a formula that transfers across related universes/periods
  (evidence against pure overfitting) is not automatically economically
  tradeable — it still needs cost modeling, sizing, and monitoring like
  any other signal.
- This is a research-process methodology, not investment advice, and does
  not itself establish that any mined formula constitutes a real,
  tradeable edge — that determination still requires the full validation
  pipeline covered elsewhere in this repo.

## Sources consulted

Recent (2026) academic and research work on LLM-driven and evolutionary
alpha mining (evolutionary frameworks that mutate/recombine candidate
formulas, agentic closed-loop factor discovery, reinforcement-fine-tuned
alpha discovery, and explicit concern in this literature about alpha decay
and noise sensitivity given non-stationary financial data). Described here
in original wording; no code or text copied from any source, and no
specific reported performance figures from individual papers are endorsed
as reproducible.
